# Domain pack: AI alignment

The first moursey.learn domain pack — and the one the harness was dogfooded on while being built.

**Goal it serves:** become an AI alignment researcher (hireable at a frontier lab), including from a
non-traditional / non-CS background.

**Stance:** you don't need to be a fluent engineer — AI writes code. You need to become a fluent
*experimentalist-thinker*: someone who knows what experiment to run, can read papers and code
without drowning, can interpret a result, and can tell when a claim is wrong. Alignment uniquely
rewards clear thinking about values, agency, honesty, and deception — so non-traditional backgrounds
are an edge.

**Contents:**
- `knowledge-map.md` — the competency graph (tiers 0→4 + cross-cutting), dependency-ordered.
- `field-pulse.md` — a dated currency snapshot to seed the Expert layer (refresh it; the frontier
  moves monthly).

**Epistemics for this field (for the Expert layer):** rank primary lab/eval-org sources
(Anthropic, METR, Apollo, AISI) and the field's venues above arXiv preprints (directional) above
secondary explainers; discount hype/SEO sites. Tell *canon vs. frontier vs. noise* — in a fast field
most new work is noise.

**Portfolio orientation:** every strand should terminate in a public artifact — a sharp write-up, a
mini-eval, a paper replication, a small novel experiment. Frontier labs hire on demonstrated, public
work far more than credentials.
